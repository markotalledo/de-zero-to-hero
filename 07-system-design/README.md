# Phase 5: System Design (continuo, 52 weeks)

> 1 problema de system design por semana (cada domingo). README + diagrama + tradeoffs + decisión + estimaciones (QPS, storage, bandwidth).

## 🗓 Cubierta como bloque "Domingo" en TODAS las semanas

Cada domingo el syllabus te da un problema específico (TinyURL, Twitter timeline, Uber matching, etc.). Los problemas escalan en complejidad:

| Semanas | Tipo de problema |
|---|---|
| 1-13 (Q1) | Sistemas simples: URL shortener, Pastebin, KV store, Rate limiter, Notification system |
| 14-26 (Q2) | Sistemas medianos: Payment system, WhatsApp, Spotify recommendations, Drive sync, Netflix billing |
| 27-39 (Q3) | Sistemas complejos: Distributed message queue, fraud detection, real-time analytics, RTB |
| 40-52 (Q4) | Capstone-level: ChatGPT memory, Copilot infra, Discord, A/B testing platform, distributed file system |

## 🛠 Output esperado (52 problemas)

Para cada uno:
- `XX-problem-name/README.md` con problem statement + diagrama
- `XX-problem-name/architecture.png` (Excalidraw)
- `XX-problem-name/tradeoffs.md` (decisiones de diseño)
